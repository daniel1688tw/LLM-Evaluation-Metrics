# LLM 評估指標 - 實現計畫

> **代理驅動執行：** 使用 superpowers:subagent-driven-development 逐任務執行。每個步驟使用複選框 (`- [ ]`) 追蹤進度。

**目標：** 實現一個全面的 LLM 評估系統，在三個任務上對預先訓練的模型進行基準測試（QA、文本摘要、RAG），並生成對比報告和指標視覺化。

**架構說明：**
- 模組化評估流程，每個任務有獨立的評估腳本
- 統一的指標收集和視覺化（使用 pandas 和 matplotlib）
- Docker 容器化以確保可重現的環境
- 結果按時間戳組織在目錄中，包含 CSV/JSON 輸出

**技術棧：**
- Python 3.9+
- Hugging Face `transformers`、`datasets`
- 評估指標：`evaluate`（ROUGE、BERTScore）、`squad` 指標、`scikit-learn`
- RAG：HippoCamp 基準
- 視覺化：pandas、matplotlib

## 全域約束

- 代碼必須在 Docker 容器中執行
- 結果必須儲存為 CSV 和 JSON 格式
- 指標以表格和圖表形式呈現
- 每個任務至少評估 2 個模型
- 提交方式：GitHub/GitLab
