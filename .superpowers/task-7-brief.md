# 任務 7：文檔編寫

## 目標
編寫全面的專案文檔，包括 README、結果總結和分析洞察。所有文檔必須使用繁體中文編寫。

## 需要建立的文件

### 1. README.md（建立）
建立主要的專案文檔。應包含：

**頂部部分：**
- 專案標題：LLM 評估指標

- 三個任務的簡述（QA、摘要、RAG）

**快速開始部分：**
- 使用 Docker 的步驟

- 本地設置步驟

  - 運行各個評估腳本的命令

**項目結構部分：**
- 目錄樹結構，說明每個文件/目錄的用途
- 結果目錄說明
- src/ 目錄各模塊說明

**配置部分：**
- 說明如何編輯 config.json
- 提供配置示例

**結果部分：**
- 評估後結果在哪裡

- 如何解讀結果

**評估指標部分：**
- **QA 指標**：Exact Match 和 F1 Score 說明
- **摘要指標**：ROUGE-1/2/L 和 BERTScore 說明
- **RAG 指標**：Hit@K、MRR、Recall 說明

**參考資源部分：**
- SQuAD 基準連結
- CNN/DailyMail 數據集連結
- ROUGE 論文連結
- BERTScore 論文連結

## 文檔寫作要求
- 所有文本必須使用**繁體中文**
- 使用清晰的層級標題（# # # 等）
- 使用列表項進行結構化信息
- 使用代碼塊展示命令
- 使用表格比較相關信息
- 提供具體的示例和數字（來自實際評估結果）

## 全域約束
- 所有文檔必須使用**繁體中文**
- 代碼示例保持英文
- 遵循 Markdown 格式
- 清晰、專業、易於理解

## 驗收標準
1. README.md 包含所有必要的部分
2. RESULTS.md 提供了結果總結和數據集說明
3. ANALYSIS.md 提供了深入的分析和洞察
4. 所有三個文檔都使用繁體中文
5. 格式正確、易於閱讀
6. 包含實際的模型和數據集信息
7. 提供了運行項目的清晰說明
